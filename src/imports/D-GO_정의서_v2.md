# D-GO (디고) 제품 정의서
> **No Data Server, Data Vault — D-GO**  
> Version 2.0 | 내부 기획 문서

---

## 1. 제품 정의

**D-GO**는 외부 서버에 데이터를 저장하지 않는 **개인용 물리적 데이터 저장소**이다.  
클라우드에 의존하지 않고, 사용자가 데이터의 생성·보관·활용 전 과정을 직접 통제하는  
**자기주권형(Self-Sovereign) 데이터 금고**를 지향한다.

---

## 2. 핵심 컨셉

| 컨셉 | 설명 |
|---|---|
| **엣지컴퓨팅 기반 폐쇄형 데이터 금고** | 외부 네트워크 미연결. 모든 처리가 장치 내부에서 완결됨 |
| **미니금고 내장** | 인감·OTP·카드·통장 등 실물 자산 병행 보관 (하이브리드형) |
| **PUF 인증** | 전용 장치와 저장소 간 반도체 지문 기반 인증. 복제 불가 |
| **데이터 암복호화** | AES-256. 도난·분실 시에도 데이터 유출 원천 차단 |
| **데이터 검사** | 유입 파일 악성코드 실시간 스캔 |
| **ID/패스워드 관리** | 중요 계정 정보 오프라인 보관 및 관리 |

---

## 3. 시스템 구성

### 3-1. D-GO Vault (하드웨어)

#### 아날로그 자산 보관
- 인감·OTP·카드·통장 등 실물 자산 수납 공간 내장

#### 디지털 자산 저장
- 이중 암호화(AES-256) 보안 스토리지
- 물리적 폐쇄망 내부 격리 보관

#### 데이터 입력

| 유형 | 입력 수단 | 비고 |
|---|---|---|
| 스캔본 | 전용 Scanner | 계약서·증명서·보안 문서 |
| 음성본 | 전용 Voice Recording Device | 회의록·녹취·메모, STT 변환 |
| 파일본 | 보안 USB | 법인 자료·기밀 파일 |
| 사진/동영상 | 보안 USB | 고유 자산 가치 멀티미디어 |

#### 데이터 출력
- 전용 보안 USB를 통한 단방향 출력

---

### 3-2. D-GO App (소프트웨어 인터페이스)

| 기능 | 설명 |
|---|---|
| **데이터 금고 관리** | 입력·출력·현황 조회 (파일은 Vault에만 존재) |
| **시점인증 연결** | SHA-256 해시 + TSA 타임스탬프 자동 발급. 법적 효력 확보 |
| **ID/패스워드 관리** | 오프라인 계정 정보 보관 |
| **법정인증 관리** | 법인 인증서 통합 관리 |
| **악성코드 분석** | 유입 파일 보안 검사 |

## 4. 확장 컨셉 (고도화 방향)

> 아래 두 가지 방향은 상호 배타적이지 않으며, 병행 또는 단계적 적용이 가능하다.

---

### 4-A. 근거리 통신 제어형 (Proximity-Lock Model)

**개념**  
D-GO Vault를 전용 앱으로만 제어되는 **근거리 통신 기반 장치**로 정의한다.  
물리적 거리 조건이 충족될 때만 파일 접근이 허용되며, 범위를 벗어나는 순간 자동으로 파일 제어가 차단된다.

**작동 원리**

```
[사용자 + 앱]  ←— 근거리 통신 (BLE / UWB) —→  [D-GO Vault]
                      ↓
         거리 ≤ 기준값  →  파일 제어 허용
         거리 > 기준값  →  파일 제어 즉시 차단 (세션 종료)
```

**핵심 특성**

| 항목 | 내용 |
|---|---|
| 통신 방식 후보 | BLE(Bluetooth Low Energy) 또는 UWB(Ultra-Wideband) |
| 거리 임계값 | 설정 가능 (예: 1m / 3m / 10m) |
| 이탈 시 동작 | 파일 잠금, 세션 종료, 접속 이력 기록 |
| 장점 | 사용자가 자리를 비운 순간 자동 보호. 사무실·공공장소 분실 시나리오에 강함 |
| 활용 시나리오 | 회의실 이탈, 자리 비움, 도난 상황의 자동 잠금 |

---

### 4-B. PUF 등록 장치 연결 제어형 (Trusted Device Binding Model)

**개념**  
PUF 칩으로 사전 등록된 특정 기기(모바일·태블릿·PC)와 **물리적으로 연결될 때만** 작동하고 제어 가능한 데이터 금고로 정의한다.  
등록되지 않은 장치로는 연결 자체가 불가하며, 연결 단절 즉시 잠금 상태로 복귀한다.

**작동 원리**

```
[PUF 등록 장치]  —— 물리적 연결 (USB-C / 도킹) ——→  [D-GO Vault]
                           ↓
    PUF 인증 성공  →  암호화 채널 활성화 + 파일 제어 허용
    미등록 장치    →  연결 거부 + 보안 이벤트 기록
    연결 해제      →  즉시 잠금 + 세션 소멸
```

**핵심 특성**

| 항목 | 내용 |
|---|---|
| 인증 기반 | PUF(Physical Unclonable Function) 반도체 지문 |
| 등록 장치 수 | 최대 N대 (기획 중, 예: 주 기기 1대 + 부 기기 2대) |
| 미인증 장치 연결 시 | 연결 거부 + 즉시 보안키 활성화 유지 (탈취 시 소멸 옵션) |
| 장점 | 장치 자체가 열쇠. 기기 분리만으로 완전한 데이터 보호 |
| 활용 시나리오 | PC·태블릿 전용 업무 환경, 법인 단말 고정 운용 |

---

### 4-A + 4-B 통합 시나리오 (Dual-Lock Model)

두 방식을 계층적으로 결합하면 **이중 잠금 구조**가 완성된다.

```
1차 조건: PUF 등록 장치와 물리적 연결 (4-B)
              ↓ 충족 시
2차 조건: 근거리 통신 거리 이내 유지 (4-A)
              ↓ 충족 시
           파일 제어 허용

→ 둘 중 하나라도 해제되면 즉시 잠금
```

> **결론:** 장치 인증(하드웨어 신뢰)과 근접 조건(물리적 존재)을 동시에 요구하는  
> 세계 최고 수준의 물리-소프트웨어 복합 보안 모델.

## 5. 보안 계층 요약

```
Layer 0  물리적 격리        — 폐쇄망, 통신 모듈 없음
Layer 1  장치 인증          — PUF 칩 기반 등록 장치만 접근 허용
Layer 2  근접 조건          — 거리 이탈 시 자동 파일 잠금 (4-A 적용 시)
Layer 3  사용자 인증        — 생체인증 / PIN
Layer 4  데이터 암호화      — AES-256 이중 암호화
Layer 5  무결성 검증        — SHA-256 + TSA 시점인증
Layer 6  악성코드 검사      — 입력 파일 실시간 스캔
Layer 7  감사 추적          — 모든 접근·반출 이력 로그 체인
```

## 6. 핵심 차별점 정리

| 기존 제품 | D-GO |
|---|---|
| 클라우드에 데이터 복사본 존재 | No Data Server. 물리적 장치에만 존재 |
| 장치 연결 시 누구나 접근 가능 | PUF 인증 장치만 연결 허용 |
| 분실 시 데이터 노출 | 보안키 자동 소멸로 데이터 보호 |
| 디지털 자산만 관리 | 아날로그 + 디지털 자산 통합 관리 |
| 원본 진위 증명 불가 | 시점인증(TSA)으로 법적 효력 확보 |
| 항상 접근 가능 (보안 취약) | 근접 조건 + 장치 인증 이중 잠금 |
