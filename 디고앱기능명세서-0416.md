# 디고(D-Go) Vault 하드웨어 및 소프트웨어 기능 명세서

## 1. 개요 (Overview)
### 1.1 목적
본 보고서는 디고(D-Go) Vault 시스템의 하드웨어 및 소프트웨어 기능을 정의 하고, 시스템 아키텍처 및 구성 요소 간의 역할을 명확히 하기 위해 작성되었다.

### 1.2 시스템 설계 원칙
본 시스템은 다음과 같은 보안 철학을 기반으로 설계된다.

### 1.3 외부 연결을 철저히 차단하는 폐쇄망 기반 구조
- 금고 본체(Server)와 전용 모바일 기기(Client) 간 암호화된 통신만 허용
- 외부 서버 개입 없는 완전 로컬 기반 보안 시스템

## 2. 시스템 아키텍처
### 2.1 전체 구조
본 시스템은 다음의 두 가지 주요 구성 요소로 이루어진다.
1. 금고 본체 (Embedded Server)
2. 모바일 기기 (Client Application)

## 3. 금고 본체 (Embedded Server)
### 3.1 운영 환경
- 리눅스 기반 임베디드 OS
- 내부 서버 + 컨트롤 보드 일체형 구조
- 8인치 터치 스크린 키 패드 및 문서 관리 시스템 탑제

### 3.2 주요 역할
- 문서 관리 서버 시스템 운영
- PUF(보안 칩) 모듈 제어
- 데이터 암호화 및 저장
- 시스템 로그 기록 및 관리

## 4. 금고 본체 기능 명세
### 4.1 PUF 기반 핵심 암호 제어 (Core Logic)
#### 4.1.1 반도체 물리적 특성을 기반으로 한 고유 키 생성
- 메모리 내 휘발성 기반 키 관리
- 외부 서버 없이 내부에서 암호화/복호화 수행 (Zero-Knowledge 암호화 구조)

### 4.2 물리-디지털 장치 연동 제어
#### 4.2.1 비상 복구 시스템
- 4.2.1.1 물리 USB 키 + 하드웨어 정보 조합
- 4.2.1.2 PUF 키 재호출 또는 데이터 복구

#### 4.2.2 잠금 장치 제어
- 4.2.2.1 전자식 버튼 및 PUF 인증 기반
- 4.2.2.2 모터 드라이버를 통한 금고 개폐 제어

### 4.3 데이터 보호 및 환경 대응
- RAID 1 기반 데이터 미러링
- SSD 이중화 저장 구조
- 하드웨어 장애 대비 데이터 보호

## 5. 모바일 기기 (Client Application)
### 5.1 운영 환경
- 관리자 전용 앱
- Android / iOS 공통 개발 (Flutter 또는 React Native)

### 5.2 주요 기능
- 관리자 인증
- 대시보드 기반 상태 모니터링
- 금고 본체와 1:1 Wi-Fi 연결
- 문서 업로드 및 다운로드 기능

## 6. 클라이언트 데이터 관리 기능
### 6.1 보안 포트 및 파일 입출력 통제
- PUF 인증 시에만 USB-C 및 네트워크 드라이브 활성화
- 외부 기기 1대만 연결 허용 및 통제
- 파일 업로드/다운로드 이력 관리

### 6.2 보안 미디어 생성 및 즉시 폐기
- **보안 카메라 기능**
    - 문서 촬영 후 PDF 생성
    - OCR(광학 문자 인식) 지원
    - 서버 전송 후 모바일 데이터 즉시 삭제 (옵션)
- **보안 음성 녹취**
    - 녹취 완료 후 서버 전송
    - 클라이언트 데이터 즉시 삭제
    - 로컬 LLM 기반 녹취록 자동 생성
    - 향후 화자 식별 기능 지원 예정

### 6.3 무결성 검증 및 데이터 관리
- 데이터 위·변조 검증 기능
- 시점 인증 처리
- H/W 기반 접근 권한 분리 (상/중/하)
- **추가 기능**
    - 모바일 파일 백업 (크롬 기록, 연락처, 중요 파일)
    - 웹사이트 ID 및 패스워드 보관 기능

### 6.4 보안 등급 기반 파티션 관리
#### 6.4.1 파티션 구조
- **저등급 (C)**: 일반 저장 공간 (비암호화)
- **중등급 (D)**: 파티션 단위 암호화 (별도 비밀번호 필요)
- **고등급 (E)**: 2중 암호화 적용 (사용자 인증 + 생체 인증 필요)

#### 6.4.2 자동 분류 기능
- 사용자 태그 기반 파일 자동 분류
- 보안 등급별 파티션 자동 할당

## 7. 향후 연구 및 개발 계획
### 7.1 저장장치 보안
- 보드 탈착 시 NVMe SSD 데이터 파기 기술 연구

### 7.2 로컬 AI 기능 확장
- 로컬 LLM 탑재 기능 확장: 내부 데이터 검색 및 정리
- Local NotebookLM 기능 (위키형 정보 시스템)

### 7.3 하드웨어 확장
- 카메라 기반 인감도장 시스템 연구

### 7.4 통신 및 추적 기능
- 독립 통신 보드(Wi-Fi, BLDC, GPS) 장착
- 전원 차단 시 신호 전송 (Heartbeat)
- 지정 클라이언트로 알림 전달
- 위치 추격 기능 검토

## 8. 결론
디고(D-Go) Vault 시스템은 폐쇄망 기반 보안 + PUF 기반 암호화 + 로컬 인공지능을 핵심으로 하는 고난도 보안 데이터 관리 시스템으로 승화 시키는 것이다. 향후 연구를 통해, AI 기반 기능 및 물리 보안 기술을 결합하여 차세대 보안 금고 플랫폼으로 확장하는 것을 목표로 한다.
